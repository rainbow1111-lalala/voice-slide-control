# 🎙️ 语音翻页器 · Voice Slide Control

> [English](README.md)

**讲到哪，翻到哪。** 说一句口令，幻灯片自动翻页 —— 不用点击器，不用回头看电脑。

你的声音，就是翻页器。

支持 **PowerPoint · WPS · PDF · 网页幻灯片（如 Gamma 导出的 HTML）**。

---

## 它怎么工作

```
你说话  →  识别口令「下一页」  →  模拟按一下方向键  →  幻灯片前进
 🎙️            ☁️ / 🔒                  ⌨️                  ▶️
```

关键在第三步：程序不去对接每个软件，而是模拟一次方向键发给当前最前面的窗口。PowerPoint、WPS、PDF、网页幻灯片在全屏放映时都用方向键翻页，所以一套程序通吃所有格式。

两种识别方式：

| | ☁️ 云端 API（推荐） | 🔒 本地模型 |
|---|---|---|
| 准确率 | 高，中文连读也稳 | 一般，短命令快读易错 |
| 联网 | 需要 | 完全离线 |
| 费用 | 用你自己的 API key（有免费额度） | 免费 |
| 下载 | 无需下载模型 | 需下载约 42MB 模型 |

## 🚀 快速开始（不用敲命令）

### 第 1 步 · 下载

GitHub 页面右上角 **Code → Download ZIP**，下载后解压。

### 第 2 步 · 双击启动

- **macOS**：双击 **`start.command`**
  - 首次被系统拦下就**右键 → 打开 → 再点打开**（只需一次）
- **Windows**：双击 **`start.bat`**

第一次启动自动准备环境（约 1-2 分钟，仅此一次），然后自动打开控制台 `http://127.0.0.1:5001`。

### 第 3 步 · 拿一个免费 API Key（推荐 Groq）

> 想完全离线、不用 key？看 [本地离线模式](#-本地离线模式)。

1. 打开 **https://console.groq.com/keys** 注册（免费）
2. 点 **Create API Key**，复制（形如 `gsk_...`）
3. 回到控制台：选 **Groq** → 粘贴 key → 点 **测试连接**
4. 看到 ✅ 就点 **保存配置**

也支持 **OpenAI**：把服务商切到 OpenAI，填 `sk-...`，追求准确选 `gpt-4o-transcribe` 模型。

### 第 4 步 · 授权（很重要）

为了「听见」你的声音、替你「按方向键」，需要授权：

**macOS** —— 系统设置 → 隐私与安全性：
- 🎤 **麦克风** → 勾选「终端」
- ♿ **辅助功能** → 勾选「终端」（模拟按键必需）

> 授权后在控制台点一次「停止」再「开始监听」让它生效。

**Windows** —— 设置 → 隐私和安全性 → 麦克风 → 允许桌面应用使用。（Windows 模拟按键无需额外授权。）

### 第 5 步 · 开讲

1. 点 **开始监听**
2. 打开 PPT / PDF，进入**全屏放映**，让窗口保持在最前
3. 讲完一页说 **「下一页」**（回退说 **「上一页」** 或 **「返回」**）

右侧实时日志会显示它听到了什么、有没有翻页。

## 🧭 控制台用法

全部点选完成，无需改文件：识别方式、API key（选服务商/模型/测试连接）、口令（标签增删，监听中实时热更新）、语言/麦克风/翻页键/灵敏度/反应速度、保存与开始监听 + 实时日志。界面双语，默认英文，右上角 中/EN 切换。

## 🔒 本地离线模式

不想联网、不想用 API key，用本地 Vosk 小模型（离线、免费）。模型约 42MB，**没有打包进下载包**（免得用云端的人白白多下）。

1. 控制台切到 **本地模型**，若没有模型，点 **⬇️ 下载中文模型**（一键，不用碰命令行）。
2. 或手动下载：
   ```bash
   cd models
   curl -LO https://alphacephei.com/vosk/models/vosk-model-small-cn-0.22.zip
   unzip vosk-model-small-cn-0.22.zip
   ```
   （英文换成 `vosk-model-small-en-us-0.15`）
3. 保存 → 开始监听。

⚠️ 小模型对短命令快读准确率有限，追求稳定用云端 API。

---

## 🌏 中文口令与同音字

短中文口令有同音字陷阱：下一页 和 下一夜 读音完全一样，模型常把字猜错（页/夜、张/章）。程序按**拼音**匹配口令而非按字，下一页 / 下一夜 / 夏夜 都归到同一个命令；同时保留一小段提前录音，避免第一个字的开头被切掉。

## 🛠️ 给开发者

```
voice-slide-control/
├── start.command / start.bat   # 双击启动（首次自动安装）
├── app.py                      # 本地网页控制台（Flask）
├── web/index.html              # 前端单页
├── voice_slides.py             # 核心引擎（也可命令行运行）
├── config.*.yaml               # 配置示例
└── requirements.txt
```

命令行直接跑引擎：

```bash
python -m venv .venv && .venv/bin/pip install -r requirements.txt
export GROQ_API_KEY=gsk_...
.venv/bin/python voice_slides.py --config config.api.zh.yaml --debug
.venv/bin/python voice_slides.py --list-devices
```

**自建识别服务**：任何 OpenAI 兼容的 `/audio/transcriptions` 接口都行，把 `base_url` 指过去即可。

## ❓ 排查：点了开始监听却没反应

看右侧日志：

1. **完全没有 `[rms]` 数字** → 麦克风没采到声。检查权限和设备。
2. **`[rms]` 说话时不变大** → 同上。
3. **rms 变大但没 `[FINAL]`** → 识别没返回。测 API key 或查网络；本地模式则是没识别出口令。
4. **出现 `▶ next` 但没翻页** → macOS 辅助功能权限没给，授权后重启监听。
5. **正常讲话偶尔误翻** → 换不易撞车的口令，或删过短的别名。

> 常见根因：在自己的终端 App 里启动，麦克风与辅助功能权限才能弹窗授权。
